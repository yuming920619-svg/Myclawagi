> **換成一個比較笨的內建機器人**

---

### 為什麼還是可能變差？
因為背景派工常常會有這些 trade-off：

- **上下文縮水**  
  main session 可能知道很多細節，但 sub-agent 只拿到一段任務摘要

- **互動頻率下降**  
  前景可以邊做邊問你；背景通常是一次交辦、一次回來

- **completion-only reporting**  
  delegate-runner 刻意不刷進度，這很安靜，但也代表中間修正回合比較少

- **任務描述如果寫差，錯得會很乾脆**  
  因為它會很認真執行錯誤目標

---

### 那怎麼避免品質掉？
這也是 delegate-runner 裡面會強調的幾件事：

- **模型要挑對**
  - routine coding / automation → codex
  - 深推理 / 長文 / 困難分析 → opus

- **prompt 要交代清楚**
  - goal
  - constraints
  - output format
  - 驗證方式

- **coding task 要有 verification gate**
  - lint
  - tests
  - RTL simulation / regression  
  沒過就不能算成功
